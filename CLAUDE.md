# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A self-hosted, open-source Claude Code Artifacts clone. An agent skill authors
self-contained HTML/Markdown pages; a Cloudflare Worker (Hono + D1 + R2) stores
them and serves them in a sandboxed viewer with strict CSP and an opaque origin.
Hosted instance at coda0.com. MIT licensed.

## Commands

```bash
pnpm dev          # wrangler dev — local state persists in .wrangler/state
pnpm test         # Worker integration tests (cloudflare vitest pool, needs workerd) — tests/worker/**
pnpm test:cli     # skill CLI tests (plain node vitest) — tests/cli/**  (NOT interchangeable with `pnpm test`)
pnpm typecheck    # runs BOTH tsconfig.json (Worker) and tsconfig.cli.json (CLI/skill) — both must pass
pnpm check        # biome check . (lint + format)
pnpm run deploy   # wrangler deploy — direct to prod. No CI gates; review a PR first.
```

Run a single Worker test: `pnpm test -- tests/worker/<file>.test.ts`.
Run a single CLI test: `pnpm test:cli -- tests/cli/<file>.test.ts`.

## Generated files — do not hand-edit

- `src/generated/` (`fonts.ts`, `marked-source.ts`) — regenerated by `scripts/vendor-fonts.mjs` / `scripts/vendor-marked.mjs`.
- `worker-configuration.d.ts` — regenerated by `pnpm cf-typegen` (wrangler types). ~549k.

## Architecture constraints

- **Strict CSP on the viewer**: artifacts render under `sandbox allow-scripts; default-src 'none'` with an opaque origin. No external requests, no storage, no `fetch`. Any viewer-side data (e.g. a version picker) must be **inlined at serve time** — runtime fetching is impossible inside the sandbox.
- **Run-worker-first**: `wrangler.jsonc` sets `run_worker_first: ["/", "/api/*", "/a/*", "/og/*", "/fonts/*", "/vendor/*"]` even though `public/` is a static assets dir — the Worker intercepts those routes for the coda0 rebrand, the `/fonts` proxy, the `/vendor/*` self-hosted mermaid bundle (served with a correct JS MIME + nosniff), and dynamic paths. Don't assume `public/` is served as-is for them.
- **D1 schema auto-applies** on first request — no separate migration step. Local dev state lives in `.wrangler/state`.
- **Max artifact content size: 4 MiB** (default; override per instance with the `MAX_CONTENT_MIB` env var).
- Bindings: D1 = `DB`, R2 = `CONTENT`.

## Two typecheck / test targets — keep them straight

`tsconfig.json` + `tests/worker/` target the Worker (cloudflare pool, workerd).
`tsconfig.cli.json` + `tests/cli/` target the skill CLI scripts under `skills/using-open-artifacts/scripts/` (node env, Node16 resolution).
Code in `skills/using-open-artifacts/scripts/` is type-checked by `tsconfig.cli.json`, not the Worker config.

## `skills/` is a published artifact, not a scratch dir

`skills/using-open-artifacts/` is the installable agent skill (shipped via `npx skills add`). It owns its own `SKILL.md`, `scripts/`, `references/` (design.md, canvas.md, recipe.md + recipe.schema.json, motion.md, interaction.md, tokens.css, deployment.md, icons.md), and `examples/recipes/`. Treat it as a published package: edits should align with its recipe schema and reference conventions, not ad hoc.

This is distinct from `.claude/skills/` (local project skills loaded by Claude Code) and `.git-agent/` (a separate tool's per-repo state — not part of the app).

## Design principles (when touching viewer chrome)

Per `@PRODUCT.md`: quiet, systems-grade chrome (Figma canvas / Linear toolbar as references). Single `--accent`, restrained color, tokens from `skills/using-open-artifacts/references/tokens.css`. **Both themes always** — the viewer stamps `data-theme`; every chrome element must read in light and dark. **Keyboard is first-class** — focusable with visible rings, no animation on high-frequency actions. The runtime is the design system: fixes land in `references/canvas.md`, never as per-artifact patches. Banned: decorative motion, orchestrated load sequences, the AI-slop tropes in `references/design.md`.

## Env / secrets

- `CREATE_TOKEN` (Worker secret, `npx wrangler secret put CREATE_TOKEN`) — optional; absent = open instance.
- `OPEN_ARTIFACTS_URL` / `OPEN_ARTIFACTS_TOKEN` — client-side env for the skill CLI pointing at an instance.
- `.dev.vars` — Wrangler local secrets (gitignored).
- `.artifacts/` — CLI runtime state (credentials, manifests); gitignored. `.artifacts.example/` ships a sample `config.json`.

## Conventions

- **BDD first**: behavior changes start as a `tests/features/*.feature` scenario (Given/When/Then) → RED test → GREEN code → REFACTOR. `.feature` files are excluded from biome.
- **Formatter/linter**: Biome — space indent (2), double quotes, semicolons. Don't use ESLint/Prettier.
- **Package manager**: pnpm. Use `pnpm add` / `pnpm remove` — never edit `package.json` by hand.
- **Workflow**: branch + PR into `main`; main is always deployable. Commits follow conventional format.
- **Remote**: `coda0HQ/open-artifacts` (a fork).
