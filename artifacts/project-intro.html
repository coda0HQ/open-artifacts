Open
Artifacts

An open-source, self-hosted clone of Claude Code Artifacts. Any coding agent publishes self-contained pages to a shareable URL — password-protected, auto-updating, served from a sandbox that blocks every external request. No accounts. No login. Free tier.

View source npx skills add FradSer/open-artifacts -s using-open-artifacts

01 What it does

01

Agent-native

Install a skill with npx skills add. Tell your agent "publish this as a page" — it writes the file and posts it.

02

Stable URLs

Bind an artifact to a channel; every regeneration lands on the same link as a new version.

03

Zero-knowledge

Password-protected artifacts are encrypted on your machine (PBKDF2 + AES-256-GCM). Server holds only ciphertext.

04

Auto-updates

The skill tracks which source files an artifact was built from and flags it stale when they change.

05

Sandboxed

Strict sandbox CSP — opaque origin, no external requests, no storage access. Untrusted HTML, contained.

06

OpenGraph

Full OG + Twitter Card meta with a 1200×630 PNG preview card rasterized on the edge. Links unfurl everywhere.

02 How it works

Click a step to expand.

user: "share the app's flows as a page"
The user asks the agent in natural language. The skill's SKILL.md teaches the agent when to publish.
agent writes page.html
Self-contained HTML — inline CSS/JS, no external requests — following the design guidance + token contract.
POST /api/artifacts
The skill CLI posts JSON to the Worker. With --channel <slug>, the server binds this topic to a stable URL.
Worker stores in D1 + R2
D1 holds metadata + version index + token hashes (strongly consistent). R2 holds the content body (read-after-write, free egress).
viewer opens https://…/a/<id>
The Worker wraps content in a skeleton and serves it with Content-Security-Policy: sandbox allow-scripts; default-src 'none'.

04 Architecture

Compute
Cloudflare Worker (Hono) — API at /api/*, viewer at /a/:id, OG card at /og/:id. Fits the 10 ms free-tier CPU because crypto stays client-side.
Metadata
D1 (SQLite, strongly consistent) — artifact record, per-version title/favicon/format, token & channel hashes. Schema self-migrates.
Content
R2 (object store, read-after-write) — immutable per-version bodies at content/<id>/<v>. Free egress, 10 GB free.
Identity
No accounts. 12-char crypto-random ids. One-time wt_ write token; only its hash stored. ch_ channel token binds a topic to a stable URL.
Skill
Zero-dep Node CLI (artifact.mjs) via npx skills add. Manifest + credentials in the user's repo; tokens never leave the machine.

05 Use it

install
# project scope, or -g for user scope
npx skills add FradSer/open-artifacts -s using-open-artifacts
export OPEN_ARTIFACTS_URL=https://open-artifacts.frad.workers.dev
publish → re-publish at the same URL
# first publish — binds the channel
node artifact.mjs create page.html --favicon 📊 \
  --channel app-interactions \
  --scope "user-facing interaction flows" \
  --watch "src/views/**/*.ts"
# → /a/7ujoRytqiVmz  (v1)

# later — same channel, same URL, new version
node artifact.mjs create page.html --favicon 📊 --channel app-interactions
# → /a/7ujoRytqiVmz  (v2)
password-protect
node artifact.mjs create report.html --favicon 🔒 \
  --title "Q3 Numbers" --password "correct-horse-battery"
# → unlock shell; plaintext never leaves your machine

06 Deploy your own

three commands
git clone https://github.com/FradSer/open-artifacts && cd open-artifacts
pnpm install
npx wrangler d1 create open-artifacts        # put database_id in wrangler.jsonc
npx wrangler r2 bucket create open-artifacts-content
pnpm run deploy                              # live
npx wrangler secret put CREATE_TOKEN          # optional gate

Schema self-initializes on first request. Free tier covers personal/small-team use.
