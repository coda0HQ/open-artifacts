What it is

An open-source clone of Claude Code Artifacts: an agent skill plus a Cloudflare backend. The skill composes body-only HTML from a Recipe; the Worker wraps it in a sandboxed skeleton; D1 holds metadata and tokens, R2 holds content bodies — both strongly consistent.

How publishing works

A zero-dep Node CLI (artifact.mjs) is the agent's only tool. Every artifact is generated from a versioned JSON Recipe plus ordered fragments; the CLI validates and composes in memory, then sends one final publish request. Run validate/create from the project root with a project-relative Recipe path — an absolute path or a non-root cwd fails with recipe must live inside the project root.

node skills/using-open-artifacts/scripts/artifact.mjs validate \ artifacts/app-interactions.recipe.json node skills/using-open-artifacts/scripts/artifact.mjs create \ artifacts/app-interactions.recipe.json
  1. Recipe — JSON owns title, favicon, scope, watch globs, channel, level, Canvas mode, locality, and encryption policy. Fragments are ordered theme, styles, body, scripts.
  2. validate — local builder checks path traversal, duplicate or oversized fragments, external scripts/styles, remote media, network APIs, and (for Canvas) frame IDs, geometry, connectors, tours, and builder-owned controls. Writes nothing.
  3. build — composes final bytes in memory: injects tokens.css, then the theme fragment, then vendored Canvas CSS/JS when canvas: true. No timestamps or random values — the build is deterministic.
  4. publish — create sends one request; the server returns id, a one-time writeToken, and a URL. Manifest v2 records the Recipe path and Recipe/input/output hashes only after the publish succeeds.
  5. update — same id, new immutable version; reads the Recipe from Manifest v2, rebuilds and re-validates in memory, publishes once. ?v=N views history; baseVersion mismatches return 409.
  6. status — hashes watched files against the last snapshot, reports every stale artifact; ack <id> advances the baseline without republishing when the change is out of scope.

Levels & canvas

Every artifact picks one of three levels — fidelity and motion budget.

1
Typography-led document. No flashy hero, no JS state.
2
Information design. Scannable, stateful, no external requests.
3
Editorial. One orchestrated motion moment, native APIs only.

canvas: true swaps the shell — a pan/zoom plane of frames instead of a scrolling document. Composes with any level.

DOM contract: you author #canvas, #plane, .oa-frame sections, optional .oa-note and one inline svg.oa-connectors. The builder injects zoom (#zoom-in/out/fit/pct) and, when any frame has data-tour, the tour controls — authoring those ids fails validation.

Connector geometry is per-edge: straight for same-row, perpendicular control points on the row midline for A-bottom → B-top, and a shared-vertical-midline S-curve for diagonal row-to-below flow. The runtime counter-scales stroke width, so integer coords suffice.

Figma-style pointer ownership: frame labels are pan surface that tap-to-focus; in-frame fields and buttons keep the pointer; Space-drag pans from anywhere. Selection stays off the pan surface except editable fields.

Named TAP_SLOP separates taps from pans; assistive clicks still open note chips; keyboard focus on a label is an instant camera jump; 44px hit targets; prefers-reduced-motion collapses physics and tweens.

This board is level 2, canvas: true.
