Open
Artifacts

An open-source engine for Claude Code Artifacts. Any coding agent publishes self-contained pages to a shareable URL — password-protected, auto-updating, served from a sandbox that blocks every external request. Publish to the hosted coda0.com with zero setup, or self-host on your own Cloudflare account. No accounts. No login. Every artifact builds from a strict Recipe plus ordered fragments against a shared design-token contract — a scrolling document, or a Canvas of pan/zoom frames.

View source npx skills add coda0HQ/open-artifacts -s using-open-artifacts

01 What it does

Agent-native

Install a skill with npx skills add. Tell your agent "publish this as a page" — it authors a Recipe plus fragments, validates, and publishes once.

Stable URLs

Bind an artifact to a channel; every regeneration lands on the same link as a new version — even when two agents publish at once.

Zero-knowledge

Password-protected artifacts are encrypted on your machine (PBKDF2 + AES-256-GCM). Server holds only ciphertext.

Auto-updates

Manifest v2 keeps the publication snapshot; a Stop hook flags source drift at end-of-turn. Opt individual artifacts in with auto-update on, then regenerate and update, or ack to dismiss.

Sandboxed

Strict sandbox CSP — opaque origin, no external requests, no storage access. Untrusted HTML, contained.

OpenGraph

Full OG + Twitter Card meta with a 1200×630 PNG preview card rasterized on the edge. Links unfurl everywhere.

02 How it works

Click a step to expand.

user: "share the app's flows as a page"
The user asks the agent in natural language. The skill's SKILL.md teaches the agent when to publish.
agent authors a Recipe + fragments
A versioned JSON Recipe plus ordered fragments (theme, body, scripts) — inline CSS/JS, no external requests — built against the skill's design charter: a @layer token contract with accessible semantic color pairs, an installed-font specimen library, a ~90-icon inline Remix Icon subset, a native-motion pattern library, an eight-state interaction contract, an optional Canvas pan/zoom runtime (Figma-style pointer ownership: labels pan and tap-to-focus; in-frame controls keep the pointer; named TAP_SLOP; pinch rubber), and a ship gate (P0 checklist + five-dimension critique) run before publishing.
validate → compose in memory → POST
The CLI validates the Recipe — rejecting external resources, malformed scripts, full-document bodies, and (for Canvas) bad frame geometry, unknown connectors, and authored zoom/tour controls — then composes final bytes in memory: injecting tokens, and for Canvas the vendored pan/zoom runtime, spotlight, tour, and zoom cluster. One publish request follows. Manifest v2 records the Recipe path and SHA-256 hashes only after the service accepts.
Worker stores in D1 + R2
D1 holds metadata + version index + token hashes (strongly consistent). R2 holds the content body (read-after-write, free egress). A ratelimits binding caps create/update abuse per colo.
viewer opens https://…/a/<id>
The Worker wraps content in a skeleton (reset, emoji favicon, viewport, light/dark data-theme toggle) and serves it with Content-Security-Policy: sandbox allow-scripts allow-modals allow-forms allow-popups; default-src 'none'; connect-src 'none'; form-action 'none'; base-uri 'none' — an opaque origin: artifact scripts can't read cookies, storage, or the API; X-Content-Type-Options: nosniff and Referrer-Policy: no-referrer round it out.

04 Architecture

Compute
Cloudflare Worker (Hono) — API at /api/*, viewer at /a/:id, OG card at /og/:id, landing and status pages at the root. Fits the 10 ms free-tier CPU because crypto stays client-side. Brand identity resolves once from the request hostname (brandFor()) and reuses everywhere the service names itself — viewer header chip, OG wordmark, not-found and invalid-version pages: coda0.com reads "coda0" and links its own root, every other host stays Open Artifacts unless its BRAND_URL is set.
Metadata
D1 (SQLite, strongly consistent) — artifact record, per-version title/favicon/format, token & channel hashes. Schema self-migrates.
Content
R2 (object store, read-after-write) — immutable per-version bodies at content/<id>/<v>. Free egress, 10 GB free.
Identity
No accounts. 12-char crypto-random ids. One-time wt_ write token; only its hash stored. ch_ channel token binds a topic to a stable URL. The token contract carries every semantic pair (--accent/--accent-on, --success/--success-on, …) for both themes, so a direction's accent reads with correct contrast in light and dark.
SaaS layer
Proposal (Phase 1). One binary, two behaviors: saasEnabled = !!(GITHUB_CLIENT_ID && GITHUB_CLIENT_SECRET && SESSION_SECRET) — secrets unset keeps every self-hoster byte-for-byte. On: GitHub OAuth (confidential client, signed-state CSRF guard), stateless HMAC session cookies (7-day, no D1/KV session writes), owner_id on artifacts, owner-scoped dashboard. Reads and the wt_/ch_ capability model stay unaffected.
Skill
Zero-dep Node CLI (artifact.mjs) via npx skills add. Manifest v2 + credentials in the user's repo; tokens never leave the machine.
Sources
Recipe + fragments. A strict JSON Recipe owns title, favicon, scope, watch globs, channel, level, Canvas mode, locality, and encryption policy. The builder validates — rejecting external resources, malformed scripts, full-document bodies, and (for Canvas) bad frame geometry, unknown connectors, and authored zoom/tour controls — composes final bytes in memory, and publishes once. Direct HTML/Markdown CLI publishing is rejected. Manifest v2 records the Recipe path and Recipe/input/output hashes after the publish succeeds.

05 Use it

install
# project scope, or -g for user scope
npx skills add coda0HQ/open-artifacts -s using-open-artifacts
# hosted instance (zero setup) — or your own self-hosted URL
export OPEN_ARTIFACTS_URL=https://coda0.com
author a Recipe, then publish
# artifacts/app-interactions.recipe.json owns title, favicon,
# scope, watch globs, channel, level, Canvas mode, locality
node artifact.mjs validate artifacts/app-interactions.recipe.json
node artifact.mjs create artifacts/app-interactions.recipe.json
# → /a/7ujoRytqiVmz  (v1)  — Manifest v2 records the path + hashes

# later — same channel, same URL, new version
node artifact.mjs update 7ujoRytqiVmz --label "refresh flows"
# → /a/7ujoRytqiVmz  (v2)  — rebuilds from the recorded Recipe in memory
auto-update opt-in
node artifact.mjs auto-update 7ujoRytqiVmz on
# installs the Stop hook if missing — running this command IS your consent
# hook now nudges only THIS artifact; every other one stays silent

node artifact.mjs status
# unaffected by the flag — always reports every stale watched artifact
password-protect
# Recipe sets security.encrypted=true + security.passwordCredential
node artifact.mjs create .artifacts/recipes.local/q3.recipe.json \
  --password "correct-horse-battery"
# → unlock shell; plaintext never leaves your machine

06 Deploy your own

three commands
git clone https://github.com/coda0HQ/open-artifacts && cd open-artifacts
pnpm install
npx wrangler d1 create open-artifacts        # put database_id in wrangler.jsonc
npx wrangler r2 bucket create open-artifacts-content
pnpm run deploy                              # live
npx wrangler secret put CREATE_TOKEN          # optional gate
pnpm dev                                     # local; state in .wrangler/state

Schema self-initializes on first request. Free tier covers personal/small-team use. BDD scenarios in tests/features/; architecture decision record in docs/architecture.md.
