Open Artifacts · self-hosted Claude Code Artifacts

架构总览

一个开源的 Claude Code Artifacts 克隆，完全跑在 Cloudflare 上：agent skill 创作 Recipe 加有序 fragments，Worker（Hono + D1 + R2）在内存里组装成最终字节，封进一个带严格 CSP 的 sandbox iframe 里服务。本文讲清楚每一层怎么搭，以及自托管时需要知道的事。

Cloudflare Worker Hono 4.12 D1 + R2 MIT coda0.com

概览

三个角色，一条直线。

整个系统只有三部分：agent skill（在用户项目里创作页面源）、Cloudflare Worker（Hono，存元数据到 D1、存内容到 R2、serve 时把内容包进带 reset 和主题切换的骨架）、以及一个沙箱 viewer（严格 CSP，opaque origin，无外部请求、无存储）。

关键张力是：要服务不可信的 HTML，又不能让它逃逸。open-artifacts 的答案是 CSP sandbox 指令——它给文档一个 opaque origin，artifact 脚本读不到 serving origin 的 cookie 和 storage，也不能 fetch 外部。connect-src 'none' 和 default-src 'none' 堵死所有外发。workerd 的 workers.dev 在 Public Suffix List 上，实例之间互相隔离。

存储选型上，KV 被否了：跨 colo 最终一致最多 60s、1 write/s/key、免费层 1000 writes/day——对频繁更新的 artifact 不合适。D1 强一致（元数据、指针），R2 binding 强一致 read-after-write（内容体）。D1 的 2MB 行上限也让 HTML 不能放 D1。

CSP 沙箱

不可信 HTML，封死。

每个 user-content 响应（viewer 页和 /raw）都带这组头。核心是 sandbox allow-scripts allow-modals allow-forms allow-popups——它给文档一个 opaque origin（等价于 iframe sandbox 的响应头版）：artifact 脚本读不到 serving origin 的 cookie / localStorage，也不能用 ambient credentials 调 API。connect-src 'none' 和 default-src 'none' 堵死所有外发。X-Content-Type-Options: nosniff 和 Referrer-Policy: no-referrer 补齐。

src/wrap.ts · contentSecurityPolicy() 
Content-Security-Policy: sandbox allow-scripts allow-modals allow-forms allow-popups;
  default-src 'none'; script-src 'unsafe-inline'; style-src 'unsafe-inline';
  img-src data: blob:; font-src data:; media-src data: blob:;
  connect-src 'none'; form-action 'none'; base-uri 'none'
X-Content-Type-Options: nosniff
Referrer-Policy: no-referrer

沙箱内无 localStorage、无 fetch、无外部子资源。任何 viewer 端数据（版本选择器、token）必须在 serve 时由 Worker 内联——运行时拿不到。

路由与 run_worker_first

public/ 不是原样直出。

wrangler.jsonc 对 /、/api/*、/a/*、/og/* 设了 run_worker_first——Worker 先于静态资源跑，拦截这些路由做 coda0 rebrand 和动态路径。landing page 是 public/ 里的静态资源，但 Worker 在托管主机（coda0.com）上用 HTMLRewriter 把品牌 token 重写成 "coda0"（src/home.ts，服务端改 HTML，crawler 和 no-JS 访客也看到 SaaS 身份）；其他 deploy 原样返回 asset。

wrangler.jsonc 
{
  "main": "src/index.ts",
  "assets": { "directory": "./public", "binding": "ASSETS",
              "run_worker_first": ["/", "/api/*", "/a/*", "/og/*"] },
  "d1_databases": [{ "binding": "DB", "database_name": "open-artifacts" }],
  "r2_buckets": [{ "binding": "CONTENT", "bucket_name": "open-artifacts-content" }]
}
方法路径用途
POST/api/artifacts创建（D1 + R2）
PUT/api/artifacts/:id更新（baseVersion 冲突 → 409，force: true 覆盖）
GET/api/artifacts/:id元数据
GET/api/artifacts/:id/raw原始内容
DELETE/api/artifacts/:id删除
GET/a/:idviewer 页（?v=N 取历史版本）
GET/og/:id1200×630 PNG 预览卡（resvg 渲染）
GET/landing（托管主机上 HTMLRewriter 重品牌）

存储：D1 + R2

元数据强一致，内容体 read-after-write。

绑定：D1 = DB，R2 = CONTENT。D1 存元数据、token 哈希、版本索引；R2 存内容体，object key 是 content/<id>/<version>。加密 artifact 存的是一个 JSON envelope（v、alg、kdf、iterations、salt、iv、ciphertext，全 base64）。

D1 schema 首请求自动应用——CREATE TABLE IF NOT EXISTS 加一组 ALTER TABLE（channel_hash、versions 的 title/description/favicon/format/encrypted 列）和一条 UNIQUE INDEX，结果用一个 WeakMap<D1Database> 记忆化，失败的尝试清 memo 避免污染 isolate；没有单独的 migration 步骤。对自托管者意味着零步骤 deploy，对 vitest miniflare 测试也零配置。

D1 表结构（概念） 
artifacts(id, token_hash, channel_hash, title, description, favicon,
           format, encrypted, current_version, created_at, updated_at)
versions(artifact_id, version, label, title, description, favicon, format,
         encrypted, size, created_at)
-- R2: content/<id>/<v>  明文或加密 envelope
-- channel_hash 上的 UNIQUE INDEX 让并发首发同一 channel 只能 mint 一个 artifact
身份与鉴权（无账号）

Artifact id：12 字符 crypto-random，base58-like 字母表（不可猜、默认不列出）。Write token：wt_ + 32 random bytes base64url，create 时返回一次；只存 SHA-256(token)，用 crypto.subtle.timingSafeEqual 比较。可选实例门：CREATE_TOKEN secret 设了，POST /api/artifacts 需要它作 bearer；没设 = 开放实例（应用层无限流，仅这道 bearer 门）。可选 canonical 域：PUBLIC_URL（如 https://coda0.com）是每个生成链接的 base。

密码保护（零知识）

服务端永远看不到密码或明文。skill CLI 在本地加密：PBKDF2-HMAC-SHA256，600,000 次迭代，16-byte salt → AES-256-GCM，每版本新 12-byte IV。Viewer：Worker 服务一个可信 unlock shell（严格 CSP，ciphertext envelope 内联——connect-src 'none' 仍成立）；浏览器派生密钥、解密、用同一骨架模板包明文，注入 <iframe sandbox="allow-scripts allow-modals" srcdoc>。密码错 = AES-GCM auth 失败 → 错误信息，无内容。workerd 把 PBKDF2 上限设 100,000 次且免费 CPU 10ms，服务端做不到。

Recipe + fragment 流水线

发布从不直接吃 HTML/Markdown。

作者写一个严格 Recipe（版本化 JSON）加有序 fragments。本地 builder 验证并在内存里确定性组装成最终 HTML 或 Markdown，然后一次发布。Worker serve 时把发布体包进 doctype/head/body 骨架（minimal reset、emoji favicon、viewport meta、主题支持）。

Recipe 声明 title、favicon、level（1 simple / 2 interactive / 3 rich）、canvas、channel（固定 URL 的 kebab-case 名）、scope、watch（glob）、local、autoUpdate，以及有序 fragments：theme（CSS）、styles（CSS）、body（HTML 或 Markdown）、scripts（JS）。Manifest v2 只记发布状态和 SHA-256 哈希（Recipe、有序输入、最终输出各一个）。

builder 拒绝：未知 Recipe 键、项目/symlink 逃逸、重复或超大 fragment、CSP 不兼容资源和 API、畸形脚本、全文档 markup 在 body fragment 里、输出超服务上限。它注入 tokens（HTML），Canvas 构建额外注入 vendored CSS/JS。小构建直接组装，大构建用本地 structure/detail 两遍，两条路最后都只发一个发布请求。

Recipe 结构（简） 
{
  "version": 1,
  "artifact": {
    "favicon": "🧭", "format": "html", "level": 2, "canvas": false,
    "channel": "architecture-overview", "scope": "...", "watch": [...],
    "local": false, "autoUpdate": false
  },
  "document": { "language": "zh-Hans", "theme": "...",
    "fragments": { "theme": [...], "body": [...], "scripts": [...] } },
  "security": { "encrypted": false, "passwordCredential": null },
  "build": { "strategy": "auto" }
}

Viewer chrome

wrap.ts 的四个块。

src/wrap.ts 服务时给每个 artifact 包骨架。四个关键块：RESET_CSS（reset + --oa-* token + .oa-header sticky + .oa-md markdown prose 基线）、THEME_SCRIPT（localStorage 存主题——这是外层 viewer 页，不是沙箱 iframe 内，opaque origin 时 try/catch 容错）、LAYOUT_SCRIPT（测 header 高度写到 --oa-header-h，把作者自写的 sticky 元素顶到 header 下方；并把 header 钉到 viewport 顶部——若作者设了 body { padding-top }，会把该 padding 折进 header 首个兄弟节点的 margin-top，header 仍贴 y=0，作者的内容垂直偏移不丢）、ogCardSvg（自包含 SVG OG 卡，src/og.ts 用 @resvg/resvg-wasm 栅格化成 PNG；内嵌 Inter + Noto Sans SC 子集让 CJK 标题直接渲，只有西里尔/谚文/阿拉伯/emoji 等覆盖不到的字符才掉到 isRenderable 品牌锁型 fallback 卡）。

sticky header 高度约 2.5rem（padding .375rem、toggle 28px，收紧到对齐设计规格），测成 --oa-header-h 暴露给页面。作者的 sticky 元素设 top: var(--oa-header-h, 2.5rem) 更可靠。viewer 的主题切换盖 prefers-color-scheme 双向。

OpenGraph：每个 viewer 响应带 OG + Twitter 标签。og:image 指向 GET /og/:id，返回 1200×630 PNG（不是 SVG——Facebook、X、LinkedIn、Slack、iMessage、Discord 都拒渲 SVG og:image）。卡是自包含 SVG，@resvg/resvg-wasm 栅格化，用 bundle 时编译进来的 wasm（Worker 运行时禁止运行时编译 wasm）。内嵌 Inter（Latin）+ Noto Sans SC 子集（GB2312 汉字、假名、CJK/全角标点），所以中英混排标题能直接栅格化，不会掉到 fallback 卡。ogCardSvg()（src/wrap.ts）里的 isRenderable() 只在西里尔/谚文/阿拉伯/emoji 等 Noto 子集覆盖不到的字符时返回品牌锁型 fallback 卡——此时真实标题仍通过 og:title meta 到达爬虫。emoji favicon 故意从 OG 卡省略（resvg 渲不了彩色 emoji），但仍作页面 favicon。

限制与绑定

免费层友好的硬边界。

4 MiB 内容上限（MAX_CONTENT_BYTES）
200 / 1000 / 60 title / description / label 字符上限
1k–10M PBKDF2 迭代区间（客户端校验）
D1 / R2 绑定 DB / CONTENT

应用层没有限流——src/api.ts 里既无 per-IP/per-colo 速率计数也无 429 路径；唯一的应用层上限是 MAX_CONTENT_BYTES（4 MiB，src/domain.ts）和 MAX_BODY_BYTES（MAX_CONTENT_BYTES * 1.5 + 16 KiB，JSON 包络膨胀前先拒）。可选实例门 CREATE_TOKEN secret 设了才挡 POST /api/artifacts，否则创建完全开放。免费层余量：Workers 100k req/day，R2 1M writes/mo，D1 100k row writes/day。本地 dev 状态在 .wrangler/state。

开发工作流

两层 typecheck / test，别混。

tsconfig.json + tests/worker/ 针对 Worker（cloudflare vitest pool，workerd）。tsconfig.cli.json + tests/cli/ 针对 skill CLI（node env，Node16 resolution）。两组不可互换——pnpm test 跑 worker，pnpm test:cli 跑 CLI。pnpm typecheck 跑两套，都要过。

常用命令 
pnpm dev          # wrangler dev — 本地状态在 .wrangler/state
pnpm test         # Worker 集成测试（cloudflare pool, workerd）
pnpm test:cli     # skill CLI 测试（plain node vitest）
pnpm typecheck    # 跑两套 tsconfig，都要过
pnpm check        # biome check .（lint + format）
pnpm run deploy   # wrangler deploy — 直上 prod，先 review PR
artifact.mjs 命令 
node skills/using-open-artifacts/scripts/artifact.mjs validate <recipe>
node skills/using-open-artifacts/scripts/artifact.mjs create <recipe>
node skills/using-open-artifacts/scripts/artifact.mjs update <id> [--label]
node skills/using-open-artifacts/scripts/artifact.mjs migrate <id>
node skills/using-open-artifacts/scripts/artifact.mjs show <id>
node skills/using-open-artifacts/scripts/artifact.mjs list
node skills/using-open-artifacts/scripts/artifact.mjs status [--hook]
node skills/using-open-artifacts/scripts/artifact.mjs ack <id>
node skills/using-open-artifacts/scripts/artifact.mjs auto-update <id> on|off
node skills/using-open-artifacts/scripts/artifact.mjs install-hook

状态在用户项目里：.artifacts/manifest.json（Manifest v2，可提交，记 id、URL、版本、Recipe 路径、各 SHA-256、build strategy、watch snapshot）。.artifacts/recipes.local/、.artifacts/fragments.local/、manifest.local.json、previews、credentials 都 gitignored。.artifacts/credentials.json 存 write/channel token 和命名密码。
