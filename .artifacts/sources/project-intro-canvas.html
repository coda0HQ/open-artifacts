What it is

An open-source clone of Claude Code Artifacts: an agent skill plus a Cloudflare backend. The skill publishes body-only HTML the Worker wraps in a sandboxed skeleton; D1 holds metadata and tokens, R2 holds content bodies — both strongly consistent.

ConcernDesign
IdentityNo accounts. 12-char crypto-random ids. Creation returns a one-time write token; only its SHA-256 is stored.
StorageD1: metadata, token hashes, version index. R2: bodies at content/<id>/<v>.
ServingCSP sandbox + default-src 'none' — opaque origin, no cookies, no storage.
PasswordsPBKDF2-HMAC-SHA256 (600k iterations) + AES-256-GCM, client-side. Server stores only {salt, iv, ciphertext}.
MarkdownVendored marked inlined — encrypted and plain share one code path.

How publishing works

A zero-dep Node CLI (artifact.mjs) is the agent's only tool. Channels bind an artifact to a stable URL so the same link survives every update.

node skills/using-open-artifacts/scripts/artifact.mjs create page.html \ --favicon χ --level 2 --canvas \ --channel app-interactions \ --scope "app interaction flows" --watch "src/views/**"
  1. create — publishes; returns id, one-time writeToken, and a URL.
  2. channel — --channel <slug> binds a stable URL; a per-channel token (ch_) updates the same link across machines.
  3. update — same id, new immutable version; ?v=N views history; baseVersion mismatches return 409.
  4. status — hashes watched files against the last snapshot, reports every stale artifact.
  5. auto-update — opt-in (off by default); the Stop hook only ever acts on artifacts you turned on.

Levels & canvas

Every artifact picks one of three levels — fidelity and motion budget.

1
Typography-led document. No flashy hero, no JS state.
2
Information design. Scannable, stateful, no external requests.
3
Editorial. One orchestrated motion moment, native APIs only.

--canvas swaps the shell — a pan/zoom plane of frames instead of a scrolling document. Composes with any level.

This board is --level 2 --canvas.
